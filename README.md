## ðŸ›  Elite Tech Stack

I chose a **High-Performance "Prism" Stack** to ensure the platform feels like a native desktop OS:

- **Framework:** [Next.js 15](https://nextjs.org/) (App Router) for high-speed server-side data aggregation.
- **Data Engine:** [Zustand](https://zustand-demo.pmnd.rs/) for efficient, lightweight global state management of gossip data.
- **Visualization:** [Recharts](https://recharts.org/) for scientific-grade analytics and [React Simple Maps](https://www.react-simple-maps.io/) for geospatial HUDs.
- **Motion:** [Framer Motion](https://www.framer.com/motion/) for liquid state transitions and refractive glass animations.
- **Styling:** [Tailwind CSS v4](https://tailwindcss.com/) for a modular, utility-first "Diamond" design system.

## ðŸ“¡ Data Engineering & Innovation

To meet the Xandeum Bounty requirements, XandOS implements several custom engineering solutions:

- **pRPC Aggregator:** A custom server-side pipeline that queries multiple Xandeum seed IPs in parallel, deduplicates the results, and normalizes the data for the UI.
- **Unit Conversion Logic:** Real-time transformation of raw bytes into human-readable industrial units (Exabytes/Petabytes).
- **Neural Geo-Mapper:** A high-performance IP-prefix mapping system that identifies the geographic region of pNodes without external API overhead.
- **Relative Time Sync:** Custom logic to derive "Online/Offline" status by comparing pNode timestamps against live server time.
